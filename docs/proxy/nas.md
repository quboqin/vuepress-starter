# 家庭NAS方案

## 概述

NAS（Network Attached Storage，网络附加存储）是家庭数据中心的核心，用于文件存储、备份、媒体服务、Docker应用等。本章详细介绍NAS系统选择、硬件配置、软件部署和最佳实践。

### 选择建议 (2026)

**新手/纯小白用户：**
- **首选**：绿联、极空间
- **理由**：零技术门槛，操作极简，开箱即用

**追求生态和稳定：**
- **首选**：群晖 DSM
- **理由**：软件生态极其丰富，网上教程多，稳定性行业标杆

**技术型/进阶用户：**
- **首选**：威联通（硬件配置高）、TrueNAS（ZFS文件系统）
- **理由**：功能全面，扩展性强，可玩性高

**影音爱好者：**
- **首选**：UnRAID
- **理由**：灵活扩容，Docker生态好，专为媒体设计

## 群晖DSM配置

### 初始化设置

**1. 查找NAS：**
```
浏览器访问: http://find.synology.com
或扫描网段: http://192.168.1.x:5000
```

**2. 安装DSM系统：**
```
上传DSM系统文件(.pat)
等待安装完成（约10分钟）
```

**3. 创建存储池：**
```
存储空间管理员 → 存储池 → 新增
选择RAID类型：
- RAID 0: 性能最高，无冗余
- RAID 1: 双盘镜像，一块容错
- RAID 5: 三盘以上，一块容错
- RAID 6: 四盘以上，两块容错
- SHR: 群晖混合RAID（推荐）
```

**4. 创建存储空间：**
```
选择文件系统：
- Btrfs: 推荐，支持快照和数据完整性检查
- ext4: 传统文件系统，兼容性好
```

## TrueNAS配置

### TrueNAS CORE vs SCALE

**TrueNAS CORE（基于FreeBSD）：**
- ✅ ZFS原生支持，性能极致
- ✅ 企业级稳定性
- ❌ Docker支持较弱（需要插件）
- ❌ 硬件兼容性一般

**TrueNAS SCALE（基于Debian）：**
- ✅ 原生Docker和K8s支持
- ✅ 硬件兼容性好
- ✅ 更适合虚拟化环境
- ✅ 推荐新用户使用

### ZFS存储池配置

**创建存储池：**
```
存储 → 池 → 添加
选择VDEV类型：
- Mirror: 镜像（类似RAID 1）
- RAIDZ1: 一盘容错（类似RAID 5）
- RAIDZ2: 两盘容错（类似RAID 6）
- RAIDZ3: 三盘容错

推荐配置:
- 4盘: 2个Mirror VDEV（性能好）
- 6盘: RAIDZ2（平衡性能和容错）
- 8盘以上: 多个RAIDZ2 VDEV
```

**添加缓存和日志：**
```
L2ARC缓存: 添加SSD作为读缓存
ZIL日志: 添加SSD提升同步写性能（可选）
```

### 共享服务配置

#### SMB共享（Windows）

```
共享 → Windows (SMB)
添加共享
路径: 选择数据集
名称: 共享名称
启用: ✓
```

#### NFS共享（Linux/macOS）

```
共享 → Unix (NFS)
添加共享
路径: 选择数据集
授权网络: 192.168.1.0/24
```

### 应用部署（SCALE）

**安装应用：**
```
应用 → 可用应用
搜索应用（如Nextcloud、Jellyfin）
点击安装
配置参数（端口、存储路径）
部署应用
```

## UnRAID配置

### UnRAID特色

**校验盘系统：**
- 1个校验盘保护数据
- 数据盘可以不同容量
- 灵活添加/移除硬盘

**Docker & VM：**
- 强大的Docker支持
- 虚拟机管理（KVM）
- 社区应用市场

### 基础配置

**1. 安装UnRAID：**
```
制作启动U盘
BIOS引导U盘启动
浏览器访问 http://tower.local
```

**2. 硬盘配置：**
```
Main → Array Devices
分配硬盘角色：
- Parity: 校验盘（最大容量盘）
- Data: 数据盘
启动阵列
格式化硬盘（xfs或btrfs）
```

**3. 共享设置：**
```
Shares → Add Share
共享名称、分配策略、协议（SMB/NFS）
```

### Docker应用

**安装社区应用插件：**
```
Plugins → Install Plugin
搜索: Community Applications
安装后在Apps标签页查看应用市场
```

**推荐应用：**
- **Plex / Jellyfin**: 媒体服务器
- **Sonarr / Radarr**: 影视自动化管理
- **qBittorrent / Transmission**: 下载工具
- **Nextcloud**: 私有云

## 最佳实践

### 数据安全

**3-2-1备份原则：**
```
3份数据副本
2种不同存储介质
1份异地备份
```

**实施方案：**
```
1. NAS本地存储（主数据）
2. 外置硬盘备份（本地备份）
3. 云存储备份（异地备份）
   - 阿里云OSS
   - 腾讯云COS
   - Backblaze B2
```

### 性能优化

**网络优化：**
```
使用千兆/2.5G/万兆网络
启用巨型帧（Jumbo Frame）: MTU 9000
Link Aggregation（链路聚合）
```

**缓存优化：**
```
添加SSD读缓存（群晖/TrueNAS）
使用SSD写缓存（需RAID 1保护）
```

### 硬盘健康监控

**SMART监控：**
```
定期检查SMART数据
关注关键参数:
- Reallocated Sector Count
- Current Pending Sector
- Power-On Hours
- Temperature

群晖: 存储空间管理员 → HDD/SSD → 健康信息
TrueNAS: 存储 → 磁盘 → S.M.A.R.T.测试
```
