## 什么是机场

**"机场"**是国内网民对代理服务提供商的俗称，因早期客户端软件如Shadowsocks的图标多为纸飞机而得名。

### 机场的工作原理

```
用户设备 (手机/电脑)
    │
    │ 加密连接
    ▼
国内中转服务器 (入口)
    │
    │ 隧道加密传输
    ▼
国外服务器 (出口)
    │
    ▼
目标网站 (Google/YouTube等)
```

### 机场的核心组件

1. **入口节点**: 国内服务器，负责接收用户连接
2. **中转线路**: 将流量传输到海外
   - **公网中转**: 成本低，可能拥堵
   - **IEPL/IPLC专线**: 不过墙，延迟低，稳定性高
3. **出口节点**: 海外服务器，访问目标网站
4. **落地IP**: 最终被目标网站识别的IP地址

## 机场与VPN的区别

| 对比项 | 机场 | VPN |
|--------|------|-----|
| **工作层级** | 应用层代理 | 网络层/数据链路层 |
| **加密范围** | 按需加密 | 全流量加密 |
| **价格** | 2-30元/月 | 20-80元/月 |
| **速度** | 通常更快 | 相对稳定 |
| **隐蔽性** | 强（伪装HTTPS） | 较弱（特征明显） |
| **配置难度** | 需配置客户端 | 一键连接 |
| **适用场景** | 翻墙访问外网 | 隐私保护、企业办公 |
| **设备限制** | 通常不限设备数 | 限制同时在线数 |

### 为什么机场更适合翻墙

1. **协议优势**: 使用SS/V2Ray/Trojan等协议，流量伪装成正常HTTPS
2. **线路优化**: 专门的跨境线路，针对中国网络优化
3. **性价比高**: 价格远低于商业VPN
4. **灵活分流**: 可以设置国内外分流，不影响国内网站速度

## 选择机场的核心标准

### 1. 运营稳定性

```
评估指标：
├── 运营时长 ≥ 2年
├── 是否有实体公司
├── 是否有Telegram频道/群组
├── 客服响应速度 < 30分钟
└── 是否有跑路历史
```

### 2. 线路质量

| 线路类型 | 特点 | 推荐度 |
|----------|------|--------|
| **IEPL/IPLC专线** | 不过墙，延迟低，稳定 | ⭐⭐⭐⭐⭐ |
| **公网隧道中转** | 有加密，成本较低 | ⭐⭐⭐⭐ |
| **BGP多线入口** | 三网优化，智能选路 | ⭐⭐⭐⭐ |
| **直连节点** | 最便宜，易被墙 | ⭐⭐⭐ |

**运营商入口选择**:
- 电信用户: 选择有电信入口或BGP三网的机场
- 移动用户: 大部分机场使用移动入口，体验较好
- 联通用户: 选择有联通优化线路的机场

### 3. 流媒体解锁能力

```
解锁能力测试：
✅ Netflix (奈飞) - 最重要
✅ YouTube Premium
✅ Disney+
✅ HBO Max
✅ ChatGPT/OpenAI
✅ TikTok区域解锁
```

### 4. 性价比评估

```
计算公式：
性价比 = (月流量 × 线路质量系数) / 月价格

示例：
机场A: 100G/月，专线，¥15 = (100 × 1.5) / 15 = 10
机场B: 200G/月，中转，¥20 = (200 × 1.0) / 20 = 10
机场C: 50G/月，专线，¥10 = (50 × 1.5) / 10 = 7.5
```

### 5. 客户端支持

| 平台 | 推荐客户端 |
|------|------------|
| Windows | Clash Verge Rev, v2rayN |
| macOS | Clash Verge Rev, ClashX |
| Android | Clash Meta, v2rayNG |
| iOS | Shadowrocket, Quantumult X |
| 路由器 | Passwall, OpenClash |

## 机场使用避坑指南

### ⚠️ 绝对不能做的事

1. **不要购买长期套餐（年付以上）**
   - 机场存在跑路风险
   - 建议先月付试用
   - 满意后再考虑季付/年付

2. **不要只有一个机场**
   - 必须准备备用机场
   - 建议2-3个机场轮换使用
   - 防止完全失联

3. **不要使用不明免费机场**
   - 可能记录用户数据
   - 可能植入恶意代码
   - 可能突然跑路

4. **不要在重要账号使用机场IP**
   - 银行账号
   - 支付平台
   - 重要邮箱

### 🚨 危险信号（可能跑路）

```
警告信号：
├── 官网无法访问
├── Telegram群组突然解散
├── 客服长期不回复
├── 大量节点突然失效
├── 长期不做活动/更新
└── 价格异常低廉（1元机场）
```

### ✅ 安全使用建议

1. **注册时使用临时邮箱**
   - 保护隐私
   - 避免骚扰

2. **使用加密支付方式**
   - 虚拟货币
   - 支付宝/微信支付时注意安全

3. **定期更换订阅链接**
   - 防止订阅泄露
   - 发现异常立即重置

4. **分流配置**
   ```yaml
   # Clash分流配置示例
   rules:
     # 国内直连
     - DOMAIN-SUFFIX,cn,DIRECT
     - GEOIP,CN,DIRECT
     
     # 敏感网站走代理
     - DOMAIN-KEYWORD,google,PROXY
     - DOMAIN-KEYWORD,youtube,PROXY
     
     # 默认规则
     - MATCH,PROXY
   ```

## 机场测速与评估方法

### 常用测速工具

| 工具 | 网址 | 特点 |
|------|------|------|
| **Speedtest** | speedtest.net | 国际标准 |
| **Fast.com** | fast.com | Netflix出品 |
| **YouTube测速** | youtube.com/test | 实际视频加载 |
| **CF测速** | speed.cloudflare.com | Cloudflare节点 |

### 测速指标解读

```
测速结果分析：

下载速度：
├── > 50 Mbps: 流畅4K视频
├── 20-50 Mbps: 流畅1080P
├── 5-20 Mbps: 基础使用
└── < 5 Mbps: 体验较差

延迟：
├── < 100ms: 优秀（游戏可用）
├── 100-200ms: 良好
├── 200-300ms: 一般
└── > 300ms: 较差

抖动（Jitter）：
├── < 10ms: 稳定
└── > 30ms: 可能丢包
```

### 晚高峰测速

```bash
# 测速时间建议
最佳测速时间：20:00 - 23:00（晚高峰）
对比测速时间：02:00 - 06:00（低峰期）

评估标准：
晚高峰速度 / 低峰期速度 > 70% = 优秀
晚高峰速度 / 低峰期速度 > 50% = 良好
晚高峰速度 / 低峰期速度 < 30% = 较差
```

## 总结

### 推荐配置方案

**轻度用户（¥10-15/月）**:
- 主力: Mikasa / Tolink
- 备用: 免费节点/公益节点

**中度用户（¥20-30/月）**:
- 主力: 闪电猫 / SSRDOG
- 备用: Mikasa / AllBlue

**重度用户（¥50+/月）**:
- 主力: 花云 / WgetCloud
- 备用: YTOO / 闪电猫

**终极方案（¥100+/月）**:
- 主力: WgetCloud + 自建VPS
- 备用: 2-3个中端机场

### 最后提醒

1. **机场不是永久服务**，要有备用方案
2. **月付优先**，降低风险
3. **多机场备份**，防止失联
4. **自建节点**是最终解决方案
